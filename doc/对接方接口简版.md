# App Publish Service 对接方接口简版

## 1. 说明

- 基础路径：`/api`
- 返回格式统一为：

```json
{
  "success": true,
  "data": {},
  "message": "OK"
}
```

- 文件上传接口使用 `multipart/form-data`
- 普通接口使用 `application/json`

## 2. 常用枚举

### 应用类型

- `1` Android
- `2` iOS

### 商店类型

- `huawei`
- `xiaomi`
- `oppo`
- `vivo`
- `yingyongbao`
- `ios`
- `google`

### 账户状态

- `0` 禁用
- `1` 启用

### 发布状态

- `draft`
- `api_pending`
- `auditing`
- `pass`
- `reject`
- `offline`

## 3. 对接推荐顺序

1. 创建应用
2. 创建商店账户
3. 上传安装包
4. 提交发布
5. 查询发布记录
6. 查询发布日志

## 4. 应用接口

### 4.1 创建应用

- `POST /api/apps`

请求示例：

```json
{
  "appName": "Demo App",
  "packageName": "com.demo.app",
  "appType": 1,
  "appDescription": "演示应用",
  "status": 1
}
```

### 4.2 修改应用

- `PUT /api/apps/{appId}`

### 4.3 应用详情

- `GET /api/apps/{appId}`

### 4.4 应用列表

- `GET /api/apps?appType=1&status=1&keyword=demo`

查询参数：

- `appType` 可选
- `status` 可选
- `keyword` 可选，模糊匹配应用名称、描述、包名

### 4.5 删除应用

- `DELETE /api/apps/{appId}`

说明：

- 会级联删除应用下的账户、版本、发布记录、发布日志、token 缓存

## 5. 账户接口

账户接口都挂在应用下面。

### 5.1 新增账户

- `POST /api/apps/{appId}/store-configs`

请求示例：

```json
{
  "storeType": "huawei",
  "accountName": "main-account",
  "email": "ops@example.com",
  "phone": "13800138000",
  "clientId": "client-id",
  "clientSecret": "client-secret",
  "token": "static-token",
  "ipWhitelist": "127.0.0.1",
  "apiStatus": 1
}
```

### 5.2 修改账户

- `PUT /api/apps/{appId}/store-configs/{configId}`

### 5.3 账户详情

- `GET /api/apps/{appId}/store-configs/{configId}`

### 5.4 账户列表

- `GET /api/apps/{appId}/store-configs`

### 5.5 启用/禁用账户

- `PUT /api/apps/{appId}/store-configs/{configId}/status?apiStatus=0`

### 5.6 删除账户

- `DELETE /api/apps/{appId}/store-configs/{configId}`

## 6. 版本接口

### 6.1 上传安装包并创建版本

- `POST /api/apps/{appId}/versions/upload`
- Content-Type: `multipart/form-data`

表单字段：

- `file` 必填，APK/AAB/IPA
- `updateLog` 可选
- `expectedVersionName` 可选
- `expectedVersionCode` 可选
- `expectedReinforced` 可选

### 6.2 版本列表

- `GET /api/apps/{appId}/versions`

### 6.3 版本详情

- `GET /api/apps/{appId}/versions/{versionId}`

## 7. 发布接口

### 7.1 提交发布

- `POST /api/releases/submit`

请求示例：

```json
{
  "versionId": 1,
  "storeTypes": ["huawei", "oppo"],
  "releaseMode": "api"
}
```

### 7.2 发布记录列表

- `GET /api/releases?appId=1&storeType=huawei&releaseStatus=auditing`

### 7.3 发布记录详情

- `GET /api/releases/{releaseId}`

### 7.4 发布日志列表

- `GET /api/releases/{releaseId}/logs`

## 8. CI/CD 接口

### 8.1 上传并触发发布

- `POST /api/cicd/releases/trigger`
- Content-Type: `multipart/form-data`

表单字段：

- `appId` 必填
- `storeTypes` 必填，多个商店用逗号分隔，例如 `huawei,oppo`
- `file` 必填
- `updateLog` 可选
- `expectedVersionName` 可选
- `expectedVersionCode` 可选
- `expectedReinforced` 可选

## 9. 常见错误

### 参数错误

```json
{
  "success": false,
  "data": null,
  "message": "storeType:不能为空"
}
```

### 资源不存在

```json
{
  "success": false,
  "data": null,
  "message": "App not found"
}
```
