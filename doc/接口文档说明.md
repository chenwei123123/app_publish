# App Publish Service 接口文档说明

## 1. 基本说明

- 服务前缀：`/api`
- 数据格式：除文件上传接口外，默认使用 `application/json`
- 字符编码：`UTF-8`
- 时间格式：`yyyy-MM-dd'T'HH:mm:ss`

## 2. 统一返回结构

所有接口都返回以下结构：

```json
{
  "success": true,
  "data": {},
  "message": "OK"
}
```

字段说明：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| success | boolean | 是否成功 |
| data | object/array/null | 业务数据 |
| message | string | 返回说明 |

## 3. 错误响应

### 3.1 参数错误

HTTP 状态码：`400`

```json
{
  "success": false,
  "data": null,
  "message": "appName:不能为空"
}
```

### 3.2 资源不存在

HTTP 状态码：`404`

```json
{
  "success": false,
  "data": null,
  "message": "App not found"
}
```

### 3.3 服务异常

HTTP 状态码：`500`

```json
{
  "success": false,
  "data": null,
  "message": "具体异常信息"
}
```

## 5. 应用管理接口

### 5.1 新建应用

- 方法：`POST`
- 路径：`/api/apps`

请求体：

```json
{
  "appName": "Demo App",
  "packageName": "com.demo.app",
  "appType": 1,
  "appDescription": "演示应用",
  "copyrightNo": "COPY-001",
  "icpNo": "ICP-001",
  "appRecordNo": "APP-001",
  "privacyUrl": "https://example.com/privacy",
  "userAgreementUrl": "https://example.com/agreement",
  "status": 1,
  "storeConfigs": [
    {
      "storeType": "huawei",
      "accountName": "main-account",
      "email": "ops@example.com",
      "phone": "13800138000",
      "clientId": "client-id",
      "clientSecret": "client-secret",
      "token": "static-token",
      "ipWhitelist": "127.0.0.1",
      "apiStatus": 1
    }
  ]
}
```

请求字段：

| 字段 | 必填 | 类型 | 说明 |
| --- | --- | --- | --- |
| appName | 是 | string | 应用名称 |
| packageName | 是 | string | 包名或 bundle id |
| appType | 是 | int | 应用类型 |
| appDescription | 否 | string | 应用描述 |
| copyrightNo | 否 | string | 软著编号 |
| icpNo | 否 | string | ICP 备案号 |
| appRecordNo | 否 | string | APP 备案号 |
| privacyUrl | 否 | string | 隐私政策地址 |
| userAgreementUrl | 否 | string | 用户协议地址 |
| status | 否 | int | 应用状态，默认 1 |
| storeConfigs | 否 | array | 初始化账户列表 |

返回数据：`AppResponse`

### 5.2 修改应用

- 方法：`PUT`
- 路径：`/api/apps/{appId}`

路径参数：

| 参数 | 类型 | 说明 |
| --- | --- | --- |
| appId | long | 应用 ID |

请求体与“新建应用”一致。

### 5.3 删除应用

- 方法：`DELETE`
- 路径：`/api/apps/{appId}`

说明：

- 删除应用时会同步删除其账户配置、令牌缓存、版本记录、发布记录、发布日志。

### 5.4 应用详情

- 方法：`GET`
- 路径：`/api/apps/{appId}`

返回数据：`AppResponse`

### 5.5 应用列表查询

- 方法：`GET`
- 路径：`/api/apps`

查询参数：

| 参数 | 必填 | 类型 | 说明 |
| --- | --- | --- | --- |
| appType | 否 | int | 按应用类型筛选 |
| status | 否 | int | 按状态筛选 |
| keyword | 否 | string | 模糊匹配应用名称、应用描述、包名 |

返回数据：`AppResponse[]`

### 5.6 应用返回对象 `AppResponse`

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| id | long | 应用 ID |
| appName | string | 应用名称 |
| packageName | string | 包名 |
| appType | int | 应用类型 |
| appDescription | string | 应用描述 |
| copyrightNo | string | 软著编号 |
| icpNo | string | ICP 备案号 |
| appRecordNo | string | APP 备案号 |
| privacyUrl | string | 隐私政策地址 |
| userAgreementUrl | string | 用户协议地址 |
| status | int | 应用状态 |
| createUser | string | 创建人 |
| updateUser | string | 更新人 |
| createTime | string | 创建时间 |
| updateTime | string | 更新时间 |
| storeConfigs | array | 账户配置列表 |

## 6. 账户管理接口

账户接口都归属于某个应用。

### 6.1 新增账户

- 方法：`POST`
- 路径：`/api/apps/{appId}/store-configs`

请求体：

```json
{
  "storeType": "huawei",
  "accountName": "main-account",
  "email": "ops@example.com",
  "phone": "13800138000",
  "clientId": "client-id",
  "clientSecret": "client-secret",
  "miPublicKey": "public-key",
  "miPrivateKey": "private-key",
  "token": "static-token",
  "ipWhitelist": "127.0.0.1",
  "apiStatus": 1
}
```

请求字段：

| 字段 | 必填 | 类型 | 说明 |
| --- | --- | --- | --- |
| storeType | 是 | string | 商店类型 |
| accountName | 否 | string | 开发者账户名称 |
| email | 否 | string | 登录邮箱 |
| phone | 否 | string | 登录手机号 |
| clientId | 否 | string | clientId / accessKey |
| clientSecret | 否 | string | clientSecret / accessSecret |
| miPublicKey | 否 | string | 小米公钥 |
| miPrivateKey | 否 | string | 小米私钥 |
| token | 否 | string | 静态 token |
| ipWhitelist | 否 | string | IP 白名单，逗号分隔 |
| apiStatus | 否 | int | 账户状态，默认 1 |

返回数据：`StoreConfigResponse`

### 6.2 修改账户

- 方法：`PUT`
- 路径：`/api/apps/{appId}/store-configs/{configId}`

请求体与“新增账户”一致。

### 6.3 删除账户

- 方法：`DELETE`
- 路径：`/api/apps/{appId}/store-configs/{configId}`

说明：

- 删除账户前会先删除该账户对应的 token 缓存。

### 6.4 账户列表查询

- 方法：`GET`
- 路径：`/api/apps/{appId}/store-configs`

返回数据：`StoreConfigResponse[]`

### 6.5 账户详情

- 方法：`GET`
- 路径：`/api/apps/{appId}/store-configs/{configId}`

返回数据：`StoreConfigResponse`

### 6.6 启用或禁用账户

- 方法：`PUT`
- 路径：`/api/apps/{appId}/store-configs/{configId}/status`

查询参数：

| 参数 | 必填 | 类型 | 说明 |
| --- | --- | --- | --- |
| apiStatus | 是 | int | 0 禁用，1 启用 |

### 6.7 账户返回对象 `StoreConfigResponse`

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| id | long | 账户配置 ID |
| storeType | string | 商店类型 |
| accountName | string | 账户名称 |
| email | string | 邮箱 |
| phone | string | 手机号 |
| clientId | string | 客户端 ID |
| clientSecret | string | 客户端密钥 |
| miPublicKey | string | 小米公钥 |
| miPrivateKey | string | 小米私钥 |
| token | string | 静态 token |
| ipWhitelist | string | IP 白名单 |
| apiStatus | int | 启用状态 |
| createUser | string | 创建人 |
| updateUser | string | 更新人 |
| createTime | string | 创建时间 |
| updateTime | string | 更新时间 |

## 7. 版本管理接口

### 7.1 版本列表

- 方法：`GET`
- 路径：`/api/apps/{appId}/versions`

返回数据：`AppVersionResponse[]`

### 7.2 版本详情

- 方法：`GET`
- 路径：`/api/apps/{appId}/versions/{versionId}`

返回数据：`AppVersionResponse`

### 7.3 上传安装包并创建版本

- 方法：`POST`
- 路径：`/api/apps/{appId}/versions/upload`
- 请求类型：`multipart/form-data`

表单字段：

| 字段 | 必填 | 类型 | 说明 |
| --- | --- | --- | --- |
| file | 是 | file | APK/AAB/IPA 文件 |
| updateLog | 否 | string | 更新日志 |
| expectedVersionName | 否 | string | 期望版本名，用于校验 |
| expectedVersionCode | 否 | int | 期望版本号，用于校验 |
| expectedReinforced | 否 | boolean | 期望加固状态，用于校验 |

返回数据：`PackageUploadResponse`

### 7.4 版本返回对象 `AppVersionResponse`

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| id | long | 版本 ID |
| appId | long | 应用 ID |
| versionName | string | 版本名称 |
| versionCode | int | 版本号 |
| packageUrl | string | 包路径 |
| updateLog | string | 更新日志 |
| reinforced | boolean | 是否加固 |
| createUser | string | 创建人 |
| updateUser | string | 更新人 |
| createTime | string | 创建时间 |

## 8. 发布管理接口

### 8.1 提交发布

- 方法：`POST`
- 路径：`/api/releases/submit`

请求体：

```json
{
  "versionId": 1,
  "storeTypes": ["huawei", "oppo"],
  "releaseMode": "api"
}
```

请求字段：

| 字段 | 必填 | 类型 | 说明 |
| --- | --- | --- | --- |
| versionId | 是 | long | 版本 ID |
| storeTypes | 是 | string[] | 目标商店列表 |
| releaseMode | 否 | string | 发布模式，默认 `api` |

返回数据：`ReleaseRecordResponse[]`

### 8.2 发布记录列表

- 方法：`GET`
- 路径：`/api/releases`

查询参数：

| 参数 | 必填 | 类型 | 说明 |
| --- | --- | --- | --- |
| appId | 否 | long | 应用 ID |
| versionId | 否 | long | 版本 ID |
| storeType | 否 | string | 商店类型 |
| releaseStatus | 否 | string | 发布状态 |

### 8.3 发布记录详情

- 方法：`GET`
- 路径：`/api/releases/{releaseId}`

返回数据：`ReleaseRecordResponse`

### 8.4 发布日志列表

- 方法：`GET`
- 路径：`/api/releases/{releaseId}/logs`

返回数据：`ReleaseTaskLogResponse[]`

### 8.5 发布记录返回对象 `ReleaseRecordResponse`

| 字段 | 类型 | 说明 |

| id | long | 发布记录 ID |

| versionId | long | 版本 ID |
| storeType | string | 商店类型 |
| releaseMode | string | 发布模式 |
| releaseStatus | string | 发布状态 |
| storeReleaseId | string | 商店侧发布单号 |
| rejectReason | string | 驳回原因 |
| apiRequestLog | string | 请求日志 |
| apiResponseLog | string | 响应日志 |
| releaseTime | string | 提交时间 |

| createUser | string | 创建人 |
| updateUser | string | 更新人 |

### 8.6 发布日志返回对象 `ReleaseTaskLogResponse`

| 字段 | 类型 | 说明 |
| --- | --- | --- |

| createUser | string | 创建人 |

## 9. CI/CD 一键发布接口

### 9.1 上传并触发发布

- 方法：`POST`
- 路径：`/api/cicd/releases/trigger`
- 请求类型：`multipart/form-data`

表单字段：

| 字段 | 必填 | 类型 | 说明 |
| --- | --- | --- | --- |
| appId | 是 | long | 应用 ID |
| storeTypes | 是 | string | 商店列表，英文逗号分隔 |
| file | 是 | file | 安装包文件 |
| updateLog | 否 | string | 更新日志 |
| expectedVersionName | 否 | string | 期望版本名 |
| expectedVersionCode | 否 | int | 期望版本号 |
| expectedReinforced | 否 | boolean | 期望加固状态 |

示例：

```text
appId=1
storeTypes=huawei,oppo
```

返回数据：`ReleaseRecordResponse[]`

## 10. 常用调用顺序

### 10.1 后台管理

1. 新建应用 `POST /api/apps`
2. 新增账户 `POST /api/apps/{appId}/store-configs`
3. 查询应用列表 `GET /api/apps`
4. 查询账户列表 `GET /api/apps/{appId}/store-configs`

### 10.2 发布流程

1. 上传安装包 `POST /api/apps/{appId}/versions/upload`
2. 提交发布 `POST /api/releases/submit`
3. 查询发布记录 `GET /api/releases`
4. 查询发布日志 `GET /api/releases/{releaseId}/logs`
