# App Publish Service 接口文档说明

## 1. 基本说明

- 服务前缀：`/api`
- 默认字符编码：`UTF-8`
- 时间类型：后端使用 `LocalDateTime`
- 常规接口默认使用 `application/json`
- 文件上传接口使用 `multipart/form-data`

OpenAPI / Swagger 相关：

- Swagger UI：`/swagger-ui.html`
- OpenAPI YAML：`/openapi.yaml`
- `doc/openapi.yaml` 会在构建时复制到静态资源目录
- 项目同时启用了 springdoc

鉴权说明：

- 当 `app.jwt-auth.enabled=false` 时，接口默认不做 JWT 校验
- 当 JWT 鉴权启用后，以下路径默认放行：
  - `/swagger-ui*`
  - `/v3/api-docs*`
  - `/openapi.yaml`
- 鉴权请求头默认是 `Authentication`
- 也支持从同名 Cookie 读取 token

---

## 2. 统一返回结构

所有控制器接口统一返回 `ApiResponse<T>`：

```json
{
  "success": true,
  "data": {},
  "message": "OK"
}
```

字段说明：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| success | boolean | 是否成功 |
| data | object / array / null | 业务数据 |
| message | string | 返回消息 |

成功响应：

- `ApiResponse.success(data)`
- `ApiResponse.success(data, message)`

失败响应：

- `ApiResponse.failure(message)`

---

## 3. 错误响应

全局异常由 `GlobalExceptionHandler` 统一处理。

### 3.1 参数错误

HTTP 状态码：`400`

适用异常：

- `IllegalArgumentException`
- `ConstraintViolationException`
- `MethodArgumentNotValidException`

示例：

```json
{
  "success": false,
  "data": null,
  "message": "appName: 不能为空"
}
```

### 3.2 未找到

HTTP 状态码：`404`

适用异常：

- `NotFoundException`

示例：

```json
{
  "success": false,
  "data": null,
  "message": "App not found"
}
```

### 3.3 三方接口异常

HTTP 状态码：按 `StoreApiException.status` 返回，常见为 `502`

示例：

```json
{
  "success": false,
  "data": null,
  "message": "Oppo submit failed: errno=123, msg=xxx"
}
```

### 3.4 鉴权失败

HTTP 状态码：`401`

示例：

```json
{
  "success": false,
  "data": null,
  "message": "Authentication header or cookie is required"
}
```

### 3.5 未知异常

HTTP 状态码：`500`

示例：

```json
{
  "success": false,
  "data": null,
  "message": "具体异常信息"
}
```

## 5. 应用管理接口

控制器：`AppController`

基础路径：`/api/apps`

### 5.1 创建应用

- 方法：`POST`
- 路径：`/api/apps`

请求体：`AppUpsertRequest`

```json
{
  "appName": "Demo App",
  "packageName": "com.demo.app",
  "appType": 1,
  "appDescription": "演示应用",
  "copyrightNo": "COPY-001",
  "icpNo": "ICP-001",
  "appRecordNo": "APP-001",
  "privacyUrl": "https://example.com/privacy",
  "userAgreementUrl": "https://example.com/agreement",
  "status": 1,
  "versionCode": "100",
  "buildCode": "100"
}
```

字段说明：

| 字段 | 必填 | 类型 | 说明 |
| --- | --- | --- | --- |
| appName | 是 | string | 应用名称 |
| packageName | 是 | string | 应用包名 |
| appType | 是 | int | 1 Android / 2 iOS |
| appDescription | 否 | string | 应用描述 |
| copyrightNo | 否 | string | 软件著作权登记号 |
| icpNo | 否 | string | ICP 备案号 |
| appRecordNo | 否 | string | 应用备案号 |
| privacyUrl | 否 | string | 隐私政策地址 |
| userAgreementUrl | 否 | string | 用户协议地址 |
| status | 否 | int | 应用状态，默认 1 |
| versionCode | 否 | string | 版本号字符串 |
| buildCode | 否 | string | 构建号 |

返回：`ApiResponse<AppResponse>`

### 5.2 更新应用

- 方法：`PUT`
- 路径：`/api/apps/{appId}`

路径参数：

| 参数 | 类型 | 说明 |
| --- | --- | --- |
| appId | long | 应用 ID |

请求体与“创建应用”一致。

返回：`ApiResponse<AppResponse>`

### 5.3 查询应用列表

- 方法：`GET`
- 路径：`/api/apps`

查询参数：

| 参数 | 必填 | 类型 | 说明 |
| --- | --- | --- | --- |
| keyword | 否 | string | 按应用名称、描述或包名模糊匹配 |

返回：`ApiResponse<List<AppResponse>>`

### 5.4 查询应用详情

- 方法：`GET`
- 路径：`/api/apps/{appId}`

返回：`ApiResponse<AppDetailResponse>`

说明：

- 返回详情中包含关联版本列表 `versions`
- 返回详情中包含关联发布记录列表 `releaseRecords`

### 5.5 删除应用

- 方法：`DELETE`
- 路径：`/api/apps/{appId}`

返回：`ApiResponse<Void>`

说明：

- 删除应用时会联动删除关联版本、发布记录等数据

### 5.6 `AppResponse`

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| id | long | 应用 ID |
| appName | string | 应用名称 |
| packageName | string | 应用包名 |
| appType | int | 应用类型 |
| appDescription | string | 应用描述 |
| copyrightNo | string | 软件著作权登记号 |
| icpNo | string | ICP 备案号 |
| appRecordNo | string | 应用备案号 |
| privacyUrl | string | 隐私政策地址 |
| userAgreementUrl | string | 用户协议地址 |
| status | int | 应用状态 |
| createUser | string | 创建人 |
| updateUser | string | 更新人 |
| createTime | string | 创建时间 |
| updateTime | string | 更新时间 |
| versions | array | 关联版本列表 |
| releaseRecords | array | 关联发版记录列表 |

### 5.7 `AppDetailResponse`

当前字段与 `AppResponse` 基本一致，也包含：

- `versions`
- `releaseRecords`

## 6. 应用商店账号配置接口

控制器：`StoreConfigController`

基础路径：`/api/store-configs`

说明：

- 当前账号配置接口已不再挂在 `/api/apps/{appId}` 下
- 支持 JSON 和 multipart 两种创建 / 更新方式
- multipart 主要用于上传小米图标文件 `iconFile` 和公钥文件 `publicKeyFile`

### 6.1 创建账号配置（JSON）

- 方法：`POST`
- 路径：`/api/store-configs`
- Content-Type：`application/json`

请求体：`StoreConfigRequest`

```json
{
  "storeType": "xiaomi",
  "accountName": "main-account",
  "email": "ops@example.com",
  "phone": "13800138000",
  "clientId": "client-id",
  "clientSecret": "client-secret",
  "publicKey": "public-key",
  "privateKey": "private-key",
  "token": "static-token",
  "ipWhitelist": "127.0.0.1",
  "privacyUrl": "https://example.com/privacy",
  "icon": "data:image/png;base64,...",
  "appId": "demo-app-id",
  "apiStatus": 1
}
```

### 6.2 创建账号配置（表单）

- 方法：`POST`
- 路径：`/api/store-configs`
- Content-Type：`multipart/form-data`

表单字段与 `StoreConfigRequest` 同名，额外支持：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| iconFile | file | 小米图标文件，后端会转成 Base64 保存 |
| publicKeyFile | file | 平台公钥文件，后端会按 UTF-8 文本读取后保存，当前主要用于小米 |

### 6.3 更新账号配置（JSON）

- 方法：`PUT`
- 路径：`/api/store-configs/{configId}`
- Content-Type：`application/json`

### 6.4 更新账号配置（表单）

- 方法：`PUT`
- 路径：`/api/store-configs/{configId}`
- Content-Type：`multipart/form-data`

### 6.5 删除账号配置

- 方法：`DELETE`
- 路径：`/api/store-configs/{configId}`

### 6.6 分页查询账号配置

- 方法：`GET`
- 路径：`/api/store-configs`

查询参数：

| 参数 | 必填 | 类型 | 说明 |
| --- | --- | --- | --- |
| current | 否 | long | 当前页 |
| size | 否 | long | 每页大小 |
| pageNum | 否 | long | `current` 别名 |
| pageSize | 否 | long | `size` 别名 |
| key | 否 | string | 按商店类型、账号名、邮箱、手机号、隐私地址或应用 ID 模糊匹配 |

返回：`ApiResponse<PageResponse<StoreConfigResponse>>`

### 6.7 查询账号配置详情

- 方法：`GET`
- 路径：`/api/store-configs/{configId}`

### 6.8 更新账号配置状态

- 方法：`PUT`
- 路径：`/api/store-configs/{configId}/status`

查询参数：

| 参数 | 必填 | 类型 | 说明 |
| --- | --- | --- | --- |
| apiStatus | 是 | int | 1 启用，0 禁用 |

### 6.9 `StoreConfigRequest`

## 8. 发布管理接口

控制器：`ReleaseController`

基础路径：`/api/releases`

### 8.1 提交发版

- 方法：`POST`
- 路径：`/api/releases/submit`

请求体：`ReleaseSubmitRequest`

```json
{
  "versionId": 1,
  "storeTypes": ["huawei", "oppo"],
  "releaseMode": "api",
  "releaseType": 1
}
```

灰度发布示例：

```json
{
  "versionId": 1,
  "storeTypes": ["vivo", "huawei", "rongyao", "sanxing"],
  "releaseMode": "api",
  "releaseType": 2,
  "grayPercent": 20,
  "grayStartTime": "2026-06-30T10:00:00",
  "grayEndTime": "2026-07-07T10:00:00"
}
```

字段说明：

| 字段 | 必填 | 类型 | 说明 |
| --- | --- | --- | --- |
| versionId | 是 | long | 版本 ID |
| storeTypes | 是 | string[] | 目标商店列表 |
| releaseMode | 否 | string | 发布模式，默认 `api` |
| releaseType | 否 | long | 1 全量，2 灰度，默认 1 |
| grayPercent | 否 | long | 灰度比例，`releaseType=2` 时必填 |
| grayStartTime | 否 | string(datetime) | 灰度开始时间，`releaseType=2` 时必填 |
| grayEndTime | 否 | string(datetime) | 灰度结束时间，`releaseType=2` 时必填 |

说明：

- `releaseType=2` 时，当前只支持：
  - `vivo`
  - `huawei`
  - `rongyao`
  - `sanxing`
- 以下商店会被直接拒绝灰度提审：
  - `oppo`
  - `xiaomi`
  - `yingyongbao`

返回：`ApiResponse<List<ReleaseRecordResponse>>`

### 8.2 分页查询发版记录

- 方法：`GET`
- 路径：`/api/releases`

查询参数：

| 参数 | 必填 | 类型 | 说明 |
| --- | --- | --- | --- |
| current | 否 | long | 当前页 |
| size | 否 | long | 每页大小 |
| pageNum | 否 | long | `current` 别名 |
| pageSize | 否 | long | `size` 别名 |
| key | 否 | string | 按商店类型、应用名、包名、应用描述或版本号模糊匹配 |

返回：`ApiResponse<PageResponse<ReleaseRecordPageResponse>>`

### 8.3 查询发版详情

- 方法：`GET`
- 路径：`/api/releases/{releaseId}`

返回：`ApiResponse<ReleaseRecordResponse>`

### 8.4 根据应用 ID 查询发版记录

- 方法：`GET`

## 9. CI/CD 一键发布接口

控制器：`CicdController`

- 方法：`POST`
- 路径：`/api/cicd/releases/trigger`

| 字段 | 必填 | 类型 | 说明 |

## 11. 常见调用顺序

### 11.1 后台管理

1. 创建应用 `POST /api/apps`
2. 创建账号配置 `POST /api/store-configs`
3. 查询应用列表 `GET /api/apps`
4. 查询账号配置列表 `GET /api/store-configs`

### 11.2 手动发布流程

1. 上传安装包 `POST /api/apps/{appId}/versions/upload`
2. 提交发版 `POST /api/releases/submit`
3. 查询发版列表 `GET /api/releases`
4. 查询发版详情 `GET /api/releases/{releaseId}`

### 11.3 CI/CD 发布流程

1. 触发一键发布 `POST /api/cicd/releases/trigger`
2. 查询发版记录 `GET /api/releases`
3. 按应用查看发版记录 `GET /api/releases/appId/{appId}`
